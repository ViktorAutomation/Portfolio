Project · Document Workflows + Custom Scripting

Automated Google Doc Creation & Parsing System

An end-to-end system that turns Airtable into the single source of truth for document generation, approval, and structured content storage. Make.com handles the orchestration, custom Google Apps Script powers approval workflows inside the Doc, and approved content flows back to Airtable — closing the loop.

Make.com Google Apps Script Airtable Google Docs Slack Webhook Integration

🎯 The Challenge

The client was caught in a familiar trap: structured data lived in Airtable, but every document had to be hand-built in Google Docs, manually shared with the team, manually approved, then manually copied back into Airtable for tracking. Lots of copy-paste, lots of stale records, lots of "wait, which version is approved?"

Goals:
  • Create Google Docs directly from structured Airtable records
  • Notify the team in Slack with direct links to new documents
  • Manage content approval inside Google Docs itself
  • Automatically parse approved content and sync it back to Airtable

🧩 The Solution

A closed-loop architecture where data flows out from Airtable into a Google Doc, gets shaped and approved there, then flows back into Airtable as structured records — with Slack notifications keeping the team aligned along the way.

Airtable Main → Google Doc → Approval in Doc → Airtable Approved
↻ Approved records link back to the Main record — closing the loop.
01
Airtable as the central database
The client's existing Google Sheet was migrated into a structured Airtable base with two linked tables:
  • Main table — source records and document generation triggers
  • Approved table — final, approved content blocks linked back to their source
02
Template-based Google Doc creation
A Google Doc template was created and stored in a designated folder. In Airtable, a user ticks a checkbox to request a new document, which triggers a Make.com scenario that:
  • Creates a new Google Doc from the template
  • Inserts the relevant data from the Airtable record
  • Updates the Airtable record with the new document's link
03
Slack notifications for team visibility
The same automation sends a Slack message with the new Doc's link to a designated channel. The team is informed instantly and can start reviewing without anyone having to remember to share anything.
04
Custom Google Apps Script — document actions Custom code
Two custom scripts were added to the Google Doc template, extending Docs beyond what no-code tools can do:
Custom menu — "Add block template"
Adds a custom action menu to the Google Docs UI, letting users insert predefined text blocks into the document with one click. Ensures consistent structure and wording across every doc.
Parse & send approved blocks
Scans the document for content blocks marked with "Approved: Yes", packages only those sections, and sends them via webhook to Make.com for the next stage.
05
Make.com webhook & Airtable sync — closing the loop
Make.com receives the parsed content via webhook and:
  • Creates a new record in the Approved Airtable table
  • Links it back to the source record in the Main table
Airtable always reflects the current, approved content — no manual updates, no drift between systems.

📈 Results & Impact

  • Automated document creation from Airtable — manual effort and copy-paste errors significantly reduced.
  • Better team communication — Slack notifications put the right Doc in front of the right people instantly.
  • Structured approval workflow — only content explicitly marked approved gets pushed back into Airtable.
  • Data integrity preserved — Airtable always reflects the current approved content, no manual reconciliation needed.
  • Extensible by design — the custom Apps Script foundation can be expanded with new menu actions, validation rules, or parsing logic without rebuilding the pipeline.
