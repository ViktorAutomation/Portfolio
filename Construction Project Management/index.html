🎯 The Challenge

Responsibilities for 600+ tasks were managed in a static Excel matrix. Every time a person changed role — new CX Provider, new Owner PM, new GC PM — the team had to touch hundreds of cells, manually, across the entire matrix. The P/C/R/W/I responsibility model (Primary, Coordinator, Reviewer, Witness/Monitor, Inspector) was preserved by hand. There was no way to assign people at the role level and let the system distribute them — everything was wired person-to-task, the hard way.

Goals:
  • Assign people to roles once, distribute them automatically to all related tasks
  • Reassign responsibilities across the entire project by changing a single role assignment
  • Preserve the auditable P/C/R/W/I responsibility model on every task
  • Build a scalable engine where responsibilities are driven by role definitions, not manual cell edits

⚖️ Before vs After

The core architectural shift: decoupling roles from people. Tasks reference roles. People are assigned to roles. The automation does the wiring.

Before — Excel matrix
  • 600+ rows in a static spreadsheet
  • People wired directly to tasks, cell by cell
  • Role change = hundreds of manual edits
  • No single source of truth for "who is in which role"
  • P/C/R/W/I maintained by hand on every task
  • High error rate, hours of work per change
After — Monday + Make
  • Roles defined once on the Main Project Board
  • People assigned to roles, not to tasks
  • Role change = one edit, cascades to all 600+ tasks
  • Main Project Board is the source of truth
  • P/C/R/W/I encoded in the template, applied automatically
  • Minutes instead of hours, no manual editing required

🧩 The Solution

Two Monday.com boards plus two Make.com scenarios form the entire engine: one board defines roles per project, one board holds the task template, one scenario creates and populates execution boards, and one scenario handles role reassignment across the entire project.

01
Main Project Board — the project hub
A central Monday board with one item per project. Each item has:
  • A status column (e.g., Create project, Reassign owners) that drives automation
  • Subitems for the 10–12 key roles: Owner PM, Owner Inspectors, CX Provider, Architect, GC PM, various subcontractors and specialists
  • People/contact columns where one person is assigned per role for that project
This board is the source of truth for who fills which role on each project.
02
Project Template Board — the responsibility matrix
A preconfigured Monday board representing the full responsibility matrix — one item per task, ~600 tasks total, original task IDs from the legacy Excel matrix preserved. Role columns define which roles are involved per task and what responsibility type each carries (P/C/R/W/I). Roles are stored in a normalized structure so automations can process them efficiently.
03
Make.com — project creation & role distribution
Trigger: Project status changes to Create project on the Main Board.
  1. Creates a new project execution board from the template (all 600+ tasks included)
  2. Renames the new board to match the project name
  3. Reads role assignments from the project's subitems on the Main Board
  4. Iterates through every task and, for each role marked P/C/R/W/I, writes the right person into the corresponding owner column
  5. Completes with a fully prepared execution board where all 600+ tasks have the right people assigned, per responsibility type, with zero manual editing
04
Make.com — owner reassignment across the project The killer feature
Trigger: Project status changes to Reassign owners on the Main Board.
  1. Reads the updated role assignments on the Main Board (new Owner PM, new CX Provider, etc.)
  2. Locates the corresponding execution project board previously created from the template
  3. Scans all tasks and subitems where each role appears, in every P/C/R/W/I column
  4. Updates the people/owner columns to the new person for that role and responsibility type, everywhere it appears
One change on the Main Project Board now cascades through hundreds of records. A new CX Provider gets reassigned to every Coordinator, Reviewer, Witness, or Inspector responsibility they hold — instantly, without anyone touching the execution board.
